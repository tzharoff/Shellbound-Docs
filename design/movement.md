# Movement

Movement is the first prototype priority.

## Inspirations

Current controller feel guess:
- Mario 64
- Mario Odyssey
- PS1 momentum

This is a direction, not a final formula.

## Baseline Inputs

- Run
- Jump
- Camera control
- Shell Dash
- Slingshot
- Interact
- Pause

## Goals

- Responsive third-person movement.
- Clear momentum without feeling slippery.
- Fun traversal in empty test spaces.
- Camera that supports exploration and platforming.
- Enough expressiveness that moving around the hub is enjoyable.

## Movement Tools

### Run

The default action. Running should feel confident and readable.

### Jump

Jumping should support traversal, collectible routes, and playful shortcuts.

### Shell Dash

Shell Dash is a signature movement action. It should feel fast, funny, and useful.

Possible uses:
- Traversal burst
- Object breaking
- Enemy impact
- Shortcut chaining

### Slingshot Traversal

The slingshot can support both combat and movement. This needs prototype testing before locking its role.

Possible uses:
- Pull toward anchors
- Hit switches
- Stun enemies
- Reach collectibles

### Camera

The camera needs to support exploration, platforming, and readable combat without fighting the player.

## Open Questions

- Does Shell Dash damage enemies, break objects, or only help traversal?
- Can the player chain movement actions?
- How expressive should aerial control be?
- Is the slingshot part of movement, interaction, or both?
- What is the minimum movement set for the first prototype?
